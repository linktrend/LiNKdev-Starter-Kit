## Cloud-Only Development

This project enforces **cloud-only** Supabase usage. Local database instances are **prohibited** for production data.

### Configure Auth

Follow [this guide](https://supabase.com/docs/guides/auth/social-login/auth-github) to set up an OAuth app with GitHub and configure Supabase to use it as an auth provider.

In your Supabase project, navigate to [auth > URL configuration](https://app.supabase.com/project/_/auth/url-configuration) and set your main production URL (e.g. https://your-deployment-url.vercel.app) as the site url.

Next, in your Vercel deployment settings, add a new **Production** environment variable called `NEXT_PUBLIC_SITE_URL` and set it to the same URL. Make sure to deselect preview and development environments to make sure that preview branches and local development work correctly.

### Install dependencies

Ensure you have [pnpm](https://pnpm.io/installation) installed and run:
```bash
pnpm install
```

#### Environment Variables

Make sure your `.env.local` file contains the following variables:

```bash
# Supabase Cloud
NEXT_PUBLIC_SUPABASE_URL="https://your-project-ref.supabase.co"
NEXT_PUBLIC_SUPABASE_ANON_KEY="your-anon-key-here"
SUPABASE_SERVICE_ROLE_KEY="your-service-role-key-here"

# GitHub OAuth (for development)
SUPABASE_AUTH_EXTERNAL_GITHUB_REDIRECT_URI="https://your-project-ref.supabase.co/auth/v1/callback"
```

#### Prohibited Operations

‚ùå **Never run these locally:**
- `supabase start`
- `supabase stop` 
- `supabase db reset`
- `supabase db push`
- `supabase db pull`
- Any localhost:5432 or 127.0.0.1:5432 connections

#### Troubleshooting

If you encounter any issues:

1. **Check Cloud Status**: Verify your Supabase Cloud project is running
2. **Environment Variables**: Ensure all required env vars are set correctly
3. **MCP Tools**: Use MCP for database operations and troubleshooting
4. **Documentation**: Refer to `docs/DB_OPERATIONS.md` for migration procedures

#### Next Steps

Once you have your cloud setup running, you can:

1. **Develop**: Make changes to your application and test them against cloud
2. **Migrate**: Create and apply database migrations via MCP
3. **Deploy**: Deploy your changes through the CI/CD pipeline

For more information about cloud development with Supabase, see the
[Supabase Cloud documentation](https://supabase.com/docs/guides/cloud).
